    如何选择算法

    发现最好算法的关键环节是反复试错的迭代过程

    st=>start: Start
c1=>condition: forecast a target value|预测
op1=>operation: supervised learning
op2=>operation: unsupervised learning
e=>end: End
c2=>condition: discrete value
op3=>operation: classification
op4=>operation: regression
c3=>condition: discrete groups
op5=>operation: clustering
op6=>operation: density estimation algorithm

st->c1
c1(yes)->c2
c2(yes)->op3
c2(no)->op4
c1(no)->op2(right)
op2->c3
op4->e
op3->e
c3(yes)->op5
c3(no)->op6(right)
op5->e
op6->e

    算法学习

    Decision trees 决策树

    • Pros: Computationally cheap to use, easy for humans to understand learned results, missing values OK, can deal with irrelevant features
    • Cons: Prone to overfitting
    • Works with: Numeric values, nominal values
